# /kb/query — 查询知识库

## 前置检测

读取 `.kb/kb.json`：
- 如果文件不存在，提示"请先运行 /kb/init 初始化知识库"
- 如果文件存在但格式损坏，提示错误并建议重新初始化

---

## Step 1：选择查询模式

使用 `question` 工具：

```
问题：「你想怎么查？」
选项：
  [1] 🔍 关键词搜索 — 输入关键词，搜索所有文档
  [2] 📋 概念查询 — 从概念索引中选择一个概念，查看详情
  [3] 📅 按时间浏览 — 查看某个时间段的记录
  [4] ❓ 查看待解决问题 — 列出所有未解决的问题
  [5] ❌ 返回
```

---

## Step 2a：关键词搜索

### A. 输入关键词

```
问题：「输入要搜索的关键词：」
示例：「Lyapunov 稳定性」「误差信号」「JWT」
```

### B. 执行搜索

搜索分两层进行：

**第一层：精确匹配**

读取 `.kb/index/concepts.json`，匹配概念名称中包含关键词的条目。

**第二层：全文搜索**

使用 `grep` 工具搜索 `.kb/docs/**/*.md` 中包含关键词的文件。

### C. 展示结果

按以下格式分组展示结果：

```
┌──────────────────────────────────────────────┐
│ 🔍 搜索结果：「{keyword}」                    │
├──────────────────────────────────────────────┤
│ 🧠 概念匹配（{count} 条）                     │
│                                              │
│ [概念名] — 状态 — 所在文档                    │
│                                              │
│ 📄 文档匹配（{count} 条）                     │
│                                              │
│ [1] docs/notes/YYYY-MM-DD-xxx.md            │
│     匹配片段：...{包含关键词的上下文}...       │
│                                              │
│ [2] docs/decisions/YYYY-MM-DD-yyy.md        │
│     匹配片段：...{包含关键词的上下文}...       │
└──────────────────────────────────────────────┘
```

如果没有匹配结果：
```
未找到与「{keyword}」相关的内容。
建议：
  - 检查拼写是否正确
  - 尝试用英文或同义词搜索
  - 使用 /kb/update 补充相关知识
```

## Step 2b：概念查询

### A. 展示概念列表

读取 `.kb/index/concepts.json`，按以下格式展示：

```
🧠 概念列表（共 {total} 个）
─────────────────────────────────

[1] 误差信号         🟢 已操作化
[2] Lyapunov函数     🟡 待操作化
[3] 语义漂移         🔴 悬空
[4] PID控制器        🟢 已操作化
...

[0] 返回上一步
```

### B. 选择概念

```
问题：「选择要查看的概念（输入编号）：」
```

### C. 展示概念详情

读取概念节点，展开详情：

```
┌──────────────────────────────────────────────┐
│ 🧠 {概念名称}                                 │
│ 状态：{状态标记}                              │
├──────────────────────────────────────────────┤
│ 📝 定义                                       │
│ {operationalization 或 "暂无操作化定义"}      │
│                                              │
│ 📍 来源文档                                   │
│ {source_doc}                                 │
│                                              │
│ 🔗 关联概念                                   │
│ 上游依赖：{上游概念列表}                       │
│ 下游应用：{下游概念列表}                       │
│                                              │
│ 📅 创建时间：{created}                        │
│ 🕐 更新时间：{updated}                       │
│                                              │
│ ─────────────────────────────                 │
│  相关文档片段：                               │
│ {从 source_doc 中提取的概念相关段落，限 200 字}│
└──────────────────────────────────────────────┘
```

如果概念状态为 🔴 悬空：
```
⚠️ 注意：此概念目前缺乏操作化定义。
可能影响依赖它的上层概念或设计决策。
建议使用 /kb/update 为其补充操作化信息。
```

## Step 2c：按时间浏览

### A. 选择时间范围

```
问题：「按什么方式浏览？」
选项：
  [1] 本月
  [2] 上个月
  [3] 最近三个月
  [4] 指定月份 — 输入 YYYY-MM
  [5] 全部时间
```

### B. 展示时间线

读取 `.kb/index/timeline.json`，筛选对应条目，按月份分组：

```
📅 时间浏览 — {时间范围标签}
─────────────────────────────────

### 2025 年 6 月

[1] 06-22 📝 Lyapunov 稳定性理论（学习笔记）
[2] 06-20 🔧 PID 控制器选型决策（决策记录）
[3] 06-15 📊 Sprint 3 进度更新（进度记录）
[4] 06-10 📐 误差信号接口规范（规格文档）

### 2025 年 5 月
...
```

### C. 展开查看

```
问题：「选择要查看的文档编号，或输入 0 返回：」
```

展示选中文档的完整内容（读取对应文件）。

## Step 2d：查看待解决问题

### A. 展示问题列表

读取 `.kb/index/questions.json`，筛选 `status: "open"` 的条目，按优先级排序：

```
❓ 待解决问题（共 {count} 个）
─────────────────────────────────

🔴 高优先级（{high_count} 个）
  [1] 语义空间距离度量
      来源：docs/notes/2025-06-22-lyapunov.md
      阻塞：Lyapunov函数操作化, 误差信号定义
  [2] PID离散动作空间适配
      来源：docs/decisions/2025-06-20-pid-selection.md
      阻塞：控制器结构选型

🟡 中优先级（{medium_count} 个）
  ...

🟢 低优先级（{low_count} 个）
  ...
```

### B. 选择问题查看详情

```
问题：「选择要查看的问题编号，或输入 0 返回：」
```

展示问题的完整信息：
```
┌──────────────────────────────────────────────┐
│ ❓ {问题描述}                                 │
│ 优先级：{优先级}                              │
│ 状态：{状态}                                  │
│ 来源：{source_doc}                            │
│ 创建时间：{created}                           │
│                                              │
│ 🔗 阻塞的上下游                               │
│ 上游阻塞因素：{blocked_by}                    │
│ 下游影响：{blocks}                           │
│                                              │
│ 💡 建议                                      │
│ {根据问题类型生成建议，如"建议使用 /kb/update │
│  补充相关知识"}                               │
└──────────────────────────────────────────────┘
```

## Step 3：查询后操作

查询结果展示后，提供后续操作建议：

```
💡 后续操作建议：
  - 如需深入查看文档，使用 /kb/query 并选择「按时间浏览」
  - 如需更新或补充内容，使用 /kb/update
  - 如问题已解决，使用 /kb/update 标注问题状态
```

---

## 错误处理

| 场景 | 处理方式 |
|------|---------|
| kb.json 不存在 | 提示"请先运行 /kb/init 初始化知识库" |
| index 文件不存在 | 提示"KB 索引文件缺失，请检查 KB 状态" |
| concepts.json 损坏 | 仅搜索全文，跳过概念匹配 |
| 搜索无结果 | 提供搜索建议和补充知识提示 |
| timeline.json 损坏 | 提示"时间线索引损坏"，提供部分修复建议 |

---

## 与其他命令的数据链路

```
依赖：
  /kb/init — 提供 kb.json（确认 KB 存在）
  /kb/update — 写入 docs/**/*.md          ← 全文搜索数据源
               index/concepts.json         ← 概念匹配数据源
               index/questions.json        ← 待解决问题数据源
               index/timeline.json         ← 时间浏览数据源

产出：
  （纯只读操作，不修改任何文件）
```
