# /kb/archive — 归档过时内容

## 前置检测

读取 `.kb/kb.json`：
- 如果文件不存在，提示"请先运行 /kb/init 初始化知识库"
- 如果文件存在但格式损坏，提示错误并建议重新初始化

---

## Step 1：选择归档方式

使用 `question` 工具：

```
问题：「你想用什么方式选择要归档的内容？」
选项：
  [1] 📄 单个文档 — 从列表中选择一个文档归档
  [2] 🏷️ 按概念 — 归档与某个概念相关的所有文档
  [3] 📅 按时间 — 归档某个时间点之前的旧文档
  [4] 🔍 自动建议 — 系统扫描，建议可能过时的文档
  [5] ❌ 返回
```

## Step 2d：自动建议

### A. 系统扫描

系统根据以下规则自动识别可能过时的文档：

**规则 1：被替代的决策**

读取 `docs/decisions/` 中已被更新的决策记录（通过检查是否有更新的同类决策）。

**规则 2：长期悬空的悬空概念**

识别 `concepts.json` 中状态为 🔴 且超过 30 天未更新的概念，关联的文档可能是遗留内容。

**规则 3：过期的进度记录**

超过 3 个月前的进度记录，如果项目已过该阶段，则归档。

**规则 4：孤立的文档**

存在但未被 timeline.json 记录，或未被任何概念引用的孤立文档。

### B. 展示建议

```
🔍 系统建议归档的文档（共 {count} 篇）
─────────────────────────────────

【已被替代的决策】
[1] 2024-11-20 技术选型决策
    原因：2025-03-15 有新的技术选型决策替代了此文档

【长期悬空概念相关】
[2] 2024-10-10 语义漂移初探
    原因：关联的「语义漂移」概念状态为🔴且超过 90 天无更新

【过期进度记录】
[3] 2024-12-15 Sprint 1 进度
    原因：项目已进入验证期，此类早期进度记录可归档

─────────────────────────────────
[输入 a] 归档全部建议
[输入编号] 选择性归档（如 1,3）
[输入 0] 返回
```

## Step 3：执行归档

### 3.1 移动文件

将文档从 `docs/<type>/` 移动到 `archive/YYYY/<type>/`：

```
.archive/YYYY/<type>/<original_filename>
```

其中 YYYY 为归档年份（从文档日期提取）。

**注意**：使用文件操作工具（如 bash mv 或 PowerShell Move-Item），确保跨平台兼容。

### 3.2 添加归档元数据

在原文件头部添加归档注释（不修改内容体）：

```markdown
---
archived: true
archived_date: {ISO 8601}
archived_reason: "{reason}"
---

（原文件内容保持不变）
```

### 3.3 更新概念索引

从 `concepts.json` 中标记文档来源为"已归档"：

```json
{
  "name": "{concept_name}",
  "source_doc": "docs/notes/YYYY-MM-DD-slug.md",
  "status": "archived"
}
```

如果概念的所有来源文档都已归档，可选：保留概念记录但标记为 `status: "archived"`。

### 3.4 更新问题索引

如果文档关联了问题，检查 `questions.json`：
- 如果问题仅来自该文档，将问题状态改为 `"archived"`
- 如果问题有其他来源，保留问题

### 3.5 更新时间线

在 `timeline.json` 中标记文档为"已归档"：

```json
{
  "date": "2025-06-20",
  "type": "decision",
  "description": "PID 控制器选型决策",
  "path": "docs/decisions/2025-06-20-pid-selection.md",
  "archived": true,
  "archived_date": "{ISO 8601}",
  "archived_reason": "已被新结论替代"
}
```

## 错误处理

| 场景 | 处理方式 |
|------|---------|
| kb.json 不存在 | 提示"请先运行 /kb/init 初始化知识库" |
| 文档不存在 | 提示"文件不存在，可能已被归档"，跳过 |
| 移动文件失败 | 提示权限错误，说明需要写入 `.kb/archive/` 的权限 |
| 索引 JSON 损坏 | 提示"索引文件损坏"，建议手动修复或重建 |
| 用户取消 | 输出"已取消归档"，不产生任何变化 |

## 与其他命令的数据链路

```
依赖：
  /kb/init — 提供 kb.json（确认 KB 存在）
  /kb/update — 写入 docs/              ← 归档数据源
               index/concepts.json       ← 按概念归档的数据源
               index/questions.json      ← 问题迁移
               index/timeline.json       ← 时间过滤

产出：
  /kb/archive → archive/YYYY/           ← 冷存储，不再被检索
                index/concepts.json       ← 清理/标记
                index/questions.json      ← 迁移/标记
                index/timeline.json       ← 标记已归档
```

归档后的文档不会被 `/kb/query` 检索到，但如果需要查找归档内容：

```
💡 需要查看归档内容？
   使用 /kb/query 时指定「包括归档」选项（如果需要）
   或直接读取 .kb/archive/ 目录
```
