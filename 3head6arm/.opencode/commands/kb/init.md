# /kb/init — 初始化知识库

## 执行检测

首先检测知识库根目录 `.kb/` 是否已存在：

```
.kb/
```

## 场景分支

### 场景 A：首次初始化（`.kb/` 不存在）

执行引导式问答，共 5 步：

---

**Step 1：项目名称**

使用 `question` 工具提问：

```
问题：「这个知识库属于哪个项目？请给一个简短的名称。」
说明：用于在多处显示项目标识，建议 4-10 个字符。
默认：如果用户未输入，取当前工作目录名（`$(pwd)` 或等效获取方式）
示例：「论文研究」「前端重构」「产品规划」
```

---

**Step 2：项目类型**

```
问题：「这个项目的类型是什么？」
选项：
  [1] 学术研究
  [2] 软件工程
  [3] 个人学习
  [4] 文档写作
  [5] 其他
影响：决定后续文档模板和索引结构的选择
```

---

**Step 3：一句话描述**

```
问题：「用一句话描述项目的核心目标或要解决的问题。」
示例：「设计一个面向 AI Agent 的闭环行为控制系统」
```

---

**Step 4：核心假设（可选）**

```
问题：「有没有不可妥协的核心约束或前提？请列出。如无可跳过。」
示例：「系统失稳可用控制论形式化描述」「用户数据不得出服务」
```

---

**Step 5：当前阶段**

```
问题：「项目当前处于什么阶段？」
选项：
  [1] 探索期 — 还在收集和评估想法
  [2] 构建期 — 正在建立核心框架
  [3] 验证期 — 框架已定，在测试验证
  [4] 维护期 — 已完成，在维护和迭代
```

---

**确认摘要**

显示即将创建的内容摘要，请求确认：

```
┌──────────────────────────────────────────────┐
│ 确认创建知识库：                               │
│                                              │
│ 项目：[用户输入的名称]                         │
│ 类型：[用户选择的类型]                         │
│ 阶段：[用户选择的阶段]                         │
│ 假设：[用户列出的假设数量] 条                   │
│                                              │
│ 将创建以下结构：                               │
│ .kb/                                          │
│ ├── kb.json（元数据）                          │
│ ├── context/project.md（项目背景）              │
│ ├── context/state.md（状态摘要）               │
│ ├── context/conventions.md（写作约定）          │
│ └── index/（概念/问题/时间线索引）             │
│                                              │
│ 确认创建？ [确认] / [修改] / [取消]           │
└──────────────────────────────────────────────┘
```

用户确认后，创建完整目录结构和初始文件。

---

### 场景 B：会话恢复（`.kb/` 已存在）

执行以下步骤：

**Step 1：读取元数据**

读取 `.kb/kb.json`，提取项目基本信息。

**Step 2：加载项目背景**

读取 `.kb/context/project.md` 的完整内容。

**Step 3：加载状态摘要**

读取 `.kb/context/state.md`（如果存在）。

**Step 4：加载待解决问题（仅高优先级前 5 条）**

读取 `.kb/index/questions.json`，筛选优先级为高的前 5 条。

**Step 5：输出恢复摘要**

```
┌──────────────────────────────────────────────┐
│ 📁 项目：[project.name]                       │
│ 📍 阶段：[project.phase]                      │
│ 📊 文档总数：[统计 docs/ 下文件数]            │
│ 🧠 概念数：[concepts.json 中的概念总数]       │
│ ⚠️ 待解决问题：[问题数]                       │
│                                              │
│ 上次进度（[state.md 中的日期]）：              │
│ [state.md 中的进度摘要，限制 3 行]            │
│                                              │
│ 最紧急的待解决问题：                          │
│ [问题列表，每条一行，限制 5 条]              │
│                                              │
│ 有什么想继续的？                              │
└──────────────────────────────────────────────┘
```

**会话恢复不创建任何文件**，直接进入就绪状态。

## 创建的文件（首次初始化）

### 1. `.kb/kb.json`

```json
{
  "name": "[项目名称]",
  "type": "[项目类型]",
  "phase": "[当前阶段]",
  "created": "[ISO 8601 日期]",
  "version": "1.0",
  "conventions": {
    "filename_date_format": "YYYY-MM-DD",
    "filename_slug_case": "kebab",
    "index_encoding": "utf-8"
  }
}
```

### 2. `.kb/context/project.md`

```markdown
# 项目背景

## 项目名称
[项目名称]

## 项目类型
[项目类型]

## 核心目标
[一句话描述]

## 核心假设
[用户列出的假设，每条一行]

## 当前阶段
[当前阶段]

## 概述
（此处由用户补充项目背景的详细描述。首次创建时为空模板。）

## 已知约束
（此处记录项目的硬性约束、边界条件。首次创建时为空模板。）
```

### 3. `.kb/context/state.md`

```markdown
# 当前状态摘要

最后更新：{date}

## 进度摘要
（由 /kb/status 自动更新）

## 活跃概念
（当前正在使用的核心概念列表）

## 待解决的高优先级问题
- [ ] 问题描述
```

### 4. `.kb/context/conventions.md`

```markdown
# 知识库写作约定

## 文档类型

| 类型 | 目录 | 职责 | 包含内容 |
|------|------|------|---------|
| 学习笔记 | docs/notes/ | 记录新学到的概念、理论或技术 | 概念定义、理解、与项目关联 |
| 决策记录 | docs/decisions/ | 记录"为什么这样做" | 背景、选项对比、决策理由、后果 |
| 进度记录 | docs/progress/ | 记录当前阶段的进展 | 阶段性成果、阻塞项、下步计划 |
| 规格文档 | docs/specifications/ | 记录"做成什么样" | 接口定义、数据结构、约束条件 |

## 文件命名

```
格式：YYYY-MM-DD-<kebab-case-slug>.md
规则：
  - 日期为文档创建日期
  - slug 使用英文 3-6 词或中文 4-8 字
  - 同一天多个文档追加 -2, -3
  - 英文统一小写，用连字符分隔
```

## 操作化状态标记

| 标记 | 含义 |
|------|------|
| 🟢 已操作化 | 有明确计算/测量方法 |
| 🟡 待操作化 | 代理指标可用，但有已知误差 |
| 🔴 悬空 | 有名称但缺乏操作化定义 |

## 概念关联

新增概念时，需同时标注：
- 上游依赖（这个概念依赖哪些已有概念）
- 下游应用（这个概念服务于哪些目标）
```

## 错误处理

| 场景 | 处理方式 |
|------|---------|
| 用户取消 | 输出"已取消，知识库未创建"，不产生任何文件 |
| 目录已存在但为空 | 询问用户是"恢复"还是"重新初始化" |
| kb.json 格式损坏 | 提示错误，询问是否重新初始化 |
| 用户无写入权限 | 提示权限错误，说明需要什么权限 |

## 与其他命令的数据链路

```
/kb/init (首次) 创建：
  .kb/kb.json              ← /update 读取项目信息
  .kb/context/project.md    ← /init (恢复) 加载
  .kb/context/state.md      ← /status 更新
  .kb/context/conventions.md ← /update 参考
  .kb/index/concepts.json   ← /update 读写
  .kb/index/questions.json  ← /update 追加 / /archive 清理
  .kb/index/timeline.json   ← /update 追加 / /archive 标记
  .kb/docs/                 ← /update 写入 / /archive 移出
```
